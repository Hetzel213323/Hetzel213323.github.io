Meta case study

I built this site with AI.

Not as a gimmick. As a demonstration of what a real human-AI workflow actually looks like.

This site was built using a collaborative workflow with Claude for content strategy, structure, and code generation, and Gemini for accessibility validation and color contrast analysis. Every section was iterated through conversation, not produced in a single prompt.

That distinction matters. AI fluency is not knowing how to prompt. It is knowing when the output is right and when it needs a human to push back.

  1. 1 Planning with Claude: Defined site goals, audience, content structure, and what each section needed to accomplish. The information architecture came from genuine back-and-forth, not a template.
  2. 2 Accessibility validation with Gemini: Used Gemini to build the WCAG contrast matrix below, verifying every color combination before committing to the palette. See the results in the Design System section.
  3. 3 Code generation with Claude: Full HTML, CSS, and JavaScript generated iteratively, with accessibility, dark mode, and responsive design built in from the start, not added later.
  4. 4 Human judgment throughout: What to cut, what to keep, what felt true to the story. The AI generates options. The human makes decisions. That is the workflow worth understanding.

Design System

WCAG 2.2 Contrast Matrix.

Accessibility isn't guesswork. I built this interactive matrix to ensure every color combination on this site meets the latest WCAG 2.2 AA standards. Hover or use your keyboard to focus on combinations to see them in action.

Contrast Matrix Overview

This table compares our 10 brand colors. Row headers indicate Background Colors. Column headers indicate Text Colors. Safe combinations generally include Navy, Deep Dark, or Ink backgrounds paired with White or Off-White text.

This is a horizontally scrollable area. Use your arrow keys while focused to scroll through the full color matrix.
